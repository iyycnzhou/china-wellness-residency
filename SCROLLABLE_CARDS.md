## 🎨 滚动条样式

**自定义滚动条**：

```css
/* 滚动条高度 */
.benefits-row::-webkit-scrollbar {
  height: 6px;
}

/* 滚动条轨道 */
.benefits-row::-webkit-scrollbar-track {
  background: var(--sand);  /* 沙色背景 */
  border-radius: 3px;
}

/* 滚动条滑块 */
.benefits-row::-webkit-scrollbar-thumb {
  background: var(--secondary);  /* 金色滑块 */
  border-radius: 3px;
}

/* 悬停效果 */
.benefits-row::-webkit-scrollbar-thumb:hover {
  background: var(--secondary-dark);  /* 深金色 */
}
```

| 项目 | 尺寸 | 说明 |

| **内边距** | 1.5rem (24px) | 文字区域内边距 |

## 💡 为什么这样设计？

### 1. 最小宽度 500px

**原因**：
- ✅ 图片可以完整显示（250px 宽度）
- ✅ 文字区域有足够空间（250px 宽度）
- ✅ 避免内容拥挤

**计算**：
```
卡片宽度 = 500px
图片宽度 = 250px (50%)
文字宽度 = 250px (50%)
```

### 2. 横向滚动

**优点**：
- ✅ 桌面端充分利用宽屏
- ✅ 用户可以主动探索内容
- ✅ 保持页面紧凑
- ✅ 增加交互性

**体验**：
- 鼠标滚轮左右滚动
- 触摸板左右滑动
- 拖动滚动条

### 3. 响应式降级

**策略**：
- 大屏（> 1440px）：3 列 + 滚动
- 中屏（1024-1440px）：2 列固定
- 小屏（< 1024px）：1 列固定

**原因**：
- 桌面用户习惯滚动
- 平板用户喜欢固定布局
- 手机用户需要简洁

## ✅ 修改总结

### CSS 修改

1. **列间距缩小**
   ```css
   gap: var(--spacing-md);  /* 从 2rem 降到 1.5rem */
   ```

2. **添加滚动支持**
   ```css
   overflow-x: auto;
   padding-bottom: var(--spacing-md);
   ```

3. **自定义滚动条**
   ```css
   ::-webkit-scrollbar { height: 6px; }
   ::-webkit-scrollbar-track { background: var(--sand); }
   ::-webkit-scrollbar-thumb { background: var(--secondary); }
   ```

4. **卡片最小宽度**
   ```css
   min-width: 500px;
   flex-shrink: 0;
   ```

5. **响应式断点**
   ```css
   @media (max-width: 1440px) {
     grid-template-columns: repeat(2, 1fr);
   }
   ```
