## 🎨 设计细节

### 1. 图文比例

```
┌─────────┬─────────┐
│  图片   │  文字   │
│  50%    │  50%    │
│         │         │
│  左图右文  │  右图左文  │
└─────────┴─────────┘
```

**交错规则**：
- 奇数权益（01, 03, 05）：左图右文
- 偶数权益（02, 04, 06）：右图左文

---

### 2. 间距设计

**行间距**：`var(--spacing-xl)` = 3rem = 48px  
**列间距**：`var(--spacing-lg)` = 2rem = 32px

```
第一行
↑ 48px
第二行
```

---

### 3. 视觉层次

**从上到下**：
1. 章节标题区
2. 第一行（3 个权益）
3. 第二行（3 个权益）
4. 详情区域（如有）

## 🎬 动画效果

### 卡片悬停

```css

  transform: translateY(-4px);
  box-shadow: 0 12px 48px rgba(0, 0, 0, 0.15);
}
```

### 图片缩放

```css
.benefit-showcase-item:hover .showcase-img {
  transform: scale(1.05);
}
```

**动画参数**：
- 时长：0.5 秒
- 缓动：cubic-bezier(0.4, 0, 0.2, 1)

## 💡 Sophie 的设计思路

**先生**：

我按照您的要求重新设计了布局：

### 1. 为什么选择 2 行 x3 列？

- ✅ **视觉平衡** - 6 个权益分成两行，每行 3 个，对称美观
- ✅ **减少滚动** - 从 6 屏减少到 2 屏，用户体验更好
- ✅ **便于对比** - 用户可以同时看到多个权益
- ✅ **保持高端** - 虽然缩小了，但仍然保持 Six Senses 风格

### 2. 尺寸调整策略

- **等比缩小** - 所有元素按比例缩小 25-30%
- **保持比例** - 图片文字仍然保持 50:50
- **精简内容** - 文案保持简洁，避免拥挤

### 3. 响应式考虑

- **桌面端** - 3 列布局，充分利用宽屏
- **平板端** - 2 列布局，适中
- **移动端** - 1 列布局，保证可读性
