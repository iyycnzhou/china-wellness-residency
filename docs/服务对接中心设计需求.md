# 服务对接中心设计需求文档

## 项目背景

全球驻留权益网络平台是一个**双边市场平台**：

### 上游（服务提供者/供给方）
- 医院
- 中医诊疗机构
- 康养机构、疗养院
- 酒店、度假村
- 景区、体验场所

### 下游（国际客户/需求方）
- 国际患者
- 康养旅游人群
- 高净值度假客户
- 企业团建/奖励旅游组织者

---

## 核心定位

当前网站的"预约咨询"板块命名和功能定位太局限。

**新定位**：双边市场的**服务交易与对接中心**

**建议命名**：
- 服务对接中心 (Service Hub)
- 预约与服务 (Bookings & Services)
- 全球服务网络 (Global Service Network)

---

## 设计任务

请从全栈工程师角度分析以下四个维度：

### 1. 功能架构设计

**下游客户侧（需求方）需要哪些功能？**
- 服务搜索与筛选
- 服务详情与套餐展示
- 在线预约/询价
- 需求发布（自定义需求）
- 预约管理与订单跟踪
- 支付功能
- 评价系统

**上游服务方侧（供给方）需要哪些功能？**
- 服务入驻申请
- 服务产品发布与管理
- 预约请求处理
- 档期/库存管理
- 客户咨询回复
- 订单管理
- 收入结算

**平台管理侧需要哪些功能？**
- 服务方资质审核
- 服务 content 审核
- 交易协调与担保
- 纠纷处理
- 数据分析与报表
- 佣金管理

---

### 2. 数据库设计建议

**核心表结构**：
- `users` - 用户表（区分客户/服务方/管理员）
- `service_providers` - 服务提供方信息
- `services` - 服务产品表
- `bookings` - 预约/订单表
- `reviews` - 评价表
- `messages` - 站内消息
- `payments` - 支付记录
- `service_categories` - 服务分类

**请给出**：
- 关键字段设计
- 表关系（ER 图）
- 状态字段设计（预约状态、订单状态等）

---

### 3. 技术实现建议

**前后端页面结构**：
- 客户端页面列表
- 服务方后台页面列表
- 管理平台页面列表

**关键 API 接口设计**：
- 服务搜索接口
- 预约创建接口
- 订单管理接口
- 支付接口
- 评价接口

**状态流转设计**：
```
客户浏览 → 选择服务 → 提交预约/询价 → 服务方确认 → 支付定金 → 服务完成 → 评价
                                            ↓
                                    平台协调/担保
```

---

### 4. 用户体验设计

**双边用户的不同入口和流程**：
- 客户如何快速找到并预约服务
- 服务方如何高效管理预约

**信任机制设计**：
- 服务方资质展示
- 用户评价系统
- 平台担保机制
- 退款政策

**降低使用门槛**：
- 简化预约流程
- 多语言支持
- 客服支持
